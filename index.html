Revit Model QA Checklist

A structured quality pass to catch modeling errors and coordination risks before a model goes out for issue.

1 Model Setup & Standards Compliance
Project browser organized per BIM Execution Plan (BEP) / office standard
Shared coordinates set and matched to survey/site data
Correct Revit template and title block version used
Worksets set up correctly with no orphaned worksets
Units, project location, and phasing set correctly
File naming convention matches project/client standard
2 Model Health & File Integrity
Central file audited — critical warnings resolved (not just minor join notices)
Unused/unplaced families and unused view templates purged
In-place families minimized or replaced with loadable families
No detached or corrupted linked models
Compact/Save As performed before major milestone issue
Tip: run Audit + Purge Unused before every coordination milestone, not just at the end.
3 Worksharing & Collaboration Hygiene
"Relinquish All Mine" performed before sync — no elements left owned
No stuck editing requests or borrowed elements outstanding
Linked models set correctly as Overlay vs. Attachment as required
Links pinned in position; no unloaded or missing links at issue time
4 Element & Family Quality
No duplicate, overlapping, or stacked elements
Elements modeled at correct LOD for current project stage
Parametric families behave correctly when flexed — no geometry breaks
Shared parameters used consistently across families
5 Views, Sheets & Annotation
View templates applied consistently — no stray view-specific overrides
All sheets have correct title block, revision, and sheet number
Dimension strings and tags reference correct elements — no orphaned tags
Print/export settings checked — line weights and plot style correct before issue
6 Coordination & Final Sign-off
True North vs. Project North confirmed correct for this model's purpose
Grids and levels match across all linked/federated models
Model ready for clash detection handoff
QA reviewer sign-off recorded with date and model version
Naming and QA process aligned with ISO 19650 or client BEP requirements
